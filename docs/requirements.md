# 需求文档

## 1. 用户场景

### 1.1 日常信息消费
用户每天从多个平台获取信息：
- **B站视频** - 技术教程、知识分享、行业动态
- **微信公众号** - 深度文章、行业分析
- **小红书** - 生活技巧、产品评测
- **网页** - 技术文档、博客文章

### 1.2 痛点
1. **信息分散** - 不同平台的内容无法统一管理
2. **难以检索** - 记得看过某内容，但找不到
3. **无法关联** - 不同来源的相关内容没有连接
4. **无法沉淀** - 看过就忘，没有转化为自己的知识

### 1.3 目标
将碎片化信息转化为结构化知识库：
```
分散的信息 → 统一入库 → 智能检索 → 知识关联 → 个人知识资产
```

## 2. 功能需求

### 2.1 数据获取（已有技能支持）

| 来源 | 技能 | 功能 |
|------|------|------|
| B站视频 | bilibili-toolkit | 提取视频文稿 |
| 微信公众号 | agent-reach | 提取文章内容 |
| 小红书 | xiaohongshu-skills | 获取笔记内容 |
| 通用网页 | agent-reach | 抓取网页内容 |

### 2.2 数据入库（待开发）

**核心功能：**
1. 接收标准化内容（title, content, source 等）
2. 自动生成 AI 摘要（内容过长时）
3. 生成 Embedding 向量（用于语义搜索）
4. 自动提取/关联标签
5. 写入数据库

**调用方式：**
- 方式1：AI 调用时传入已获取的内容
- 方式2：传入 URL，自动获取并入库

### 2.3 知识检索（待开发）

**搜索方式：**
1. **关键词搜索** - 标题、内容全文匹配
2. **向量搜索** - 语义相似度匹配
3. **混合搜索** - 关键词 + 向量结合
4. **标签筛选** - 按标签过滤

**返回结果：**
- 相关内容列表
- 相似度分数
- 来源信息

### 2.4 知识关联（待开发）

- 自动推荐相关内容
- 标签关联网络
- 引用链追踪

---

## 3. 非功能需求

### 3.1 性能
- 入库速度：单条 < 5 秒（含 Embedding 生成）
- 搜索响应：< 1 秒

### 3.2 扩展性
- 支持新增来源类型
- 支持自定义 Embedding 模型
- 支持导出到其他工具（Obsidian、Notion）

### 3.3 可维护性
- 数据获取与存储解耦
- 清晰的技能边界
- 完善的文档

## 4. 用户流程

### 4.1 典型使用场景

**场景1：保存 B站视频知识**
```
用户: 帮我把这个 B站视频存到知识库
     https://www.bilibili.com/video/BV1xxx

AI: 
1. 调用 bilibili-toolkit 提取文稿
2. 调用 knowledge-skill 入库
3. 返回入库成功 + 摘要
```

**场景2：搜索历史知识**
```
用户: 我之前看过一篇关于 RAG 的文章，帮我找找

AI:
1. 调用 knowledge-skill 搜索 "RAG"
2. 向量语义匹配
3. 返回相关内容列表
```

**场景3：知识问答**
```
用户: 根据我的知识库，总结一下 AI Agent 的核心技术

AI:
1. 检索知识库中相关内容
2. 汇总整理
3. 生成回答（带引用）
```
